# 反问纪律（Grilling Discipline · 怎么问得好）

> `0-change.md` / `1-requirement.md` 的反问门决定**该不该继续问**，本文件提供**怎么问得好**的执行纪律。
> 目标：在最前端把"人机意图鸿沟"关掉，让 PROPOSAL / SPEC 建立在**问清楚**的事实上，而非 AI 假设。
>
---

## 为什么需要它

需求捕获是整条流水线**误差最大**的一环:这里没问清,后面 DESIGN / TASK / DEV 全在放大同一个误解。
"反问"本身不难,难的是**问得好**。常见失效:
- AI 甩"你觉得呢?"把认知负担推回用户
- 能 grep / Read 自答的问题也去问用户,浪费轮次
- 问题发散、跨分支拼凑,用户答得累却没答到要害
- 先问了依赖下游的问题,拿到的答案后面又推翻
- 不知道何时算问清,要么草草收场要么无限追问

本文件把反问纪律收束成 5 条硬规则 + 6 型提问模式 + 输出格式,直接执行。

---

## 五条纪律（硬规则）

### 纪律 1 · 每轮最多 3 问,且同分支

Gantry 保留批处理节奏:**每轮最多 3 个问题**,提出后等用户回答再继续。
但同一轮的 3 个问题**必须属于同一决策分支**——不跨分支硬凑数量。宁可只问 1 个,也不要把无关问题拼在一起造成信息过载。

### 纪律 2 · 每问必带推荐答案 + 一句话理由

**禁止**"你觉得呢?""你想怎么做?"这类空问。
每个问题都要给出**你的推荐立场 + 一句话理由**。用户只需确认或推翻,而不是从零构思。
推荐答案来自:代码库现状、项目 CONTEXT.md 已锁决策、领域常识、同类项目惯例。

### 纪律 3 · 先探索代码库,再问用户

能被 `grep` / `Read` 回答的问题,**先自己查**,把结论作为"我查到 X,确认?"呈现——**不占用当轮 3 个问题的名额**。
只有代码库无法回答(涉及业务意图、用户偏好、未来规划、外部约束)的,才真正抛给用户。
这条能省下大量往返:用户不该回答 AI 自己就能查到的事。

### 纪律 4 · 深度优先 + 依赖排序

走决策树时**深度优先**:一个分支问透了再开下一个,不要浅尝辄止地并行铺开。
识别依赖:若决策 B 依赖决策 A(A 的结果会改变 B 的选项),**先问 A**。
被上游决策阻塞的问题不要提前问——拿到的答案会作废。

### 纪律 5 · 明确停止条件

满足**任一**即收敛,不无限追问:
- 每个决策分支都有了答案
- 连续追问已问不出新东西
- 用户明确喊停("可以了""往下走吧")
- 累计约 15 问后收益明显递减

收敛时**输出「已锁定决策」清单**:把所有拿到的答案归纳成决策列表,作为 PROPOSAL / SPEC 的输入。
> 衔接门禁:`0-change` 收敛后,PROPOSAL 的 `## 待澄清问题` 段必须为 `无`——未决问题应继续问,不能带进下一阶段。

## 六型强制提问模式

抓不住要害时,套用这 6 型把模糊追问变成有锋芒的问题:

| 型 | 模式 | 示例 |
|---|---|---|
| **意图** | 为何选 X 而不是 Y?(点名被排除的替代方案) | "为何用轮询而不是 WebSocket?实时性要求到什么程度?" |
| **选择** | 选哪一侧?决定性约束是什么? | "服务端渲染还是客户端?决定因素是首屏还是交互复杂度?" |
| **开放** | 什么在阻塞这个决策?阻塞何时解除? | "定价模型没定会卡住配额设计——这块什么时候能定?" |
| **权衡** | 优化哪一侧?止损线在哪? | "为一致性牺牲可用性,还是反过来?什么情况下推翻这个选择?" |
| **依赖** | 上游决策锁定了吗?没锁就先解决它 | "鉴权方案还没定,这直接决定会话存储——先定鉴权吗?" |
| **不确定性** | 哪怕只有 60% 把握,你的最佳猜测是? | "不用完全确定——你现在倾向哪个?我按这个先推进,后面可改。" |

---

## 输出格式

每个问题按此格式呈现:

```
Q[序号]: <问题>
推荐:<你的立场> 因为 <一句话理由>
```

自查代码库后的确认(不占问题名额):

```
我查了代码,发现 <证据>。<据此的推断>,确认?
```

收敛时:

```
✅ 已锁定决策:
1. <决策点> → <结论>
2. ...
```

## 自检

反问过程中对照:

- [ ] 每个问题都带了推荐答案 + 理由,没有空问"你觉得呢"
- [ ] 能查代码的都先查了,没拿代码库能答的问题问用户
- [ ] 同一轮问题属于同一分支,没跨分支硬凑
- [ ] 有依赖的决策,先问了上游
- [ ] 收敛时输出了「已锁定决策」清单
